# 非機能要件一覧

| id | category | requirement | targetValue | priority |
| --- | --- | --- | --- | --- |
| NFR-001 | 性能 | SQLiteの接続タイムアウトを最適化し、スケジューラやAPIからの同時アクセス時のDBロックエラーを低減する。（変更前：デフォルトタイムアウト / 変更後：タイムアウト30.0秒を設定し、WALモードと組み合わせて耐ロック性を向上） | DBアクセス競合時のロックエラー発生率0.1%以下 | 高 |
| NFR-002 | 運用 | 日次ログローテーション時のZIP圧縮処理によるメインスレッドのブロッキングを回避または軽減する。（変更前：ロールオーバー時に同期的にZIP圧縮を実行 / 変更後：ログ出力に対する遅延影響を最小化する処理フローの最適化） | ログローテーション時のシステム応答遅延1秒未満 | 中 |
| NFR-003 | 可用性 | システムクラッシュ後の再起動時における、古いロックファイルの判定処理を強化する。（変更前：PIDファイルの単純な存在確認のみ / 変更後：Windows(ctypes)・Linux(os.kill)の各OS固有APIを用いた確実なプロセス生存確認の実装） | 異常終了後のデーモン再起動成功率100% | 高 |
| NFR-004 | 可用性 | mDNSパケット処理における自分自身のパケット再帰処理（無限ループ）を完全に防止する。（変更前：ループガードが不十分 / 変更後：QRビットの判定、送信元IPアドレスの自己判定によるパケットの早期破棄処理を追加） | ネットワーク内での不必要なマルチキャストトラフィック（自己参照ループ）の発生件数0 | 高 |
| NFR-005 | セキュリティ | 他プロキシからのレコード登録APIにおけるノード認証を強化する。（変更前：固定トークンのプレフィックスによる簡易判定 / 変更後：TokenおよびX-Sender-Node-IDヘッダーを組み合わせたプロキシ識別と認証による厳密化） | 未承認ノードからのレコード登録リクエストの遮断率100% | 中 |
| NFR-006 | 可用性 | 複数NIC（ネットワークインターフェース）環境下でのmDNSマルチキャスト送信の安定化を行う。（変更前：OSのデフォルトルーティングに依存 / 変更後：IP_MULTICAST_IF設定によりメインIPを持つインターフェースを明示的に指定） | 複雑なネットワーク環境下におけるmDNSパケットの到達失敗を削減 | 中 |
| NFR-007 | 拡張性 | 標準的なOS（Windows, Linux）およびMicroPython環境（Raspberry Pi Pico）での実行をサポートする。 | 各プラットフォームでの単一コードベースによる安定稼働 | 高 |
